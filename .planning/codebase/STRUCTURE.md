## Directory Purposes

**public/:**
- Purpose: Static assets not processed by bundler
- Contains: Images (hero section, products), favicon, SVG placeholders
- Key files: `public/images/hero/` contains all product images, hero images, family photos
- Served as-is; URLs resolve to `/images/hero/filename.ext`

**src/:**
- Purpose: All application source code
- Contains: React components, TypeScript files, styles, data
- Key files: `App.tsx` (root), `main.tsx` (entry), `index.css` (themes)

**src/components/:**
- Purpose: UI component modules
- Contains: Functional React components with TypeScript interfaces
- Key files:
  - `Hero.tsx`: Hero section with animations and CTA buttons
  - `Navigation.tsx`: Fixed header with responsive mobile menu
  - `About.tsx`: Artist bio section with decorative elements
  - `ProductCard.tsx`: Individual product card with hover effects
  - `ProductGrid.tsx`: Product grid section container
  - `ProductModal.tsx`: Full-screen product detail modal with UV toggle
  - `Footer.tsx`: Footer with branding and copyright
  - `CustomCursor.tsx`: Custom cursor implementation (if used)

**src/data/:**
- Purpose: Application data and data utilities
- Contains: Product array, interfaces, helper functions
- Key files: `products.ts` exports Product interface, products array, getProductsByCategory(), getProductById()

**src/assets/:**
- Purpose: Local SVG and media files imported by components
- Contains: React logo SVG (placeholder)
- Imported directly into components for bundling

**dist/:**
- Purpose: Bundled production output
- Generated by: `npm run build` (vite build)
- Contains: Minified JS, CSS, optimized images
- Not committed to git (.gitignore)

## Key File Locations

**Entry Points:**
- `index.html`: HTML template loaded by browser, includes root div and module script reference
- `src/main.tsx`: React app initialization, imports and renders App component
- `src/App.tsx`: Root React component, layout orchestration

**Configuration:**
- `package.json`: NPM dependencies (React, Tailwind, Framer Motion, Vite, ESLint, TypeScript)
- `vite.config.ts`: Vite bundler config with React plugin and Tailwind CSS plugin
- `tsconfig.json`: TypeScript references for app and node tsconfigs
- `tsconfig.app.json`: Compilation settings for `src/` files (target ES2022, JSX support, strict mode)
- `eslint.config.js`: ESLint rules including React hooks and React Refresh plugins

**Styling:**
- `src/index.css`: Global styles, CSS variables (--color-*), font imports, base resets
- All component styling via Tailwind utility classes with custom CSS variables

**Core Logic:**
- `src/App.tsx`: State management (menuOpen, activeCategory), product filtering, layout composition
- `src/data/products.ts`: Hardcoded product data, product interface, utility functions

**Testing:**
- Not applicable; no test files present in codebase

## Naming Conventions

**Files:**
- Components: PascalCase with `.tsx` extension (e.g., `ProductCard.tsx`)
- Data/utilities: camelCase with `.ts` extension (e.g., `products.ts`)
- Config: kebab-case with extensions (e.g., `tsconfig.app.json`, `eslint.config.js`)

**Directories:**
- Lowercase, descriptive plural nouns (components, data, assets, public)
- Special dirs: `dist` (output), `.git` (version control)

**Component Props:**
- PascalCase with `Props` suffix (e.g., `HeroProps`, `ProductCardProps`)
- Interfaces defined within component file

**Variables & Functions:**
- camelCase for variables and functions
- UPPERCASE for constants and CSS variable names (e.g., `--color-forest`)
- Prefix underscore for private/unused params (e.g., `_`)

**Import Organization:**
- React imports first: `import { useState } from 'react'`
- Third-party libraries second: `import { motion } from 'framer-motion'`
- Local imports last: `import ProductCard from './ProductCard'`

## Where to Add New Code

**New Feature (e.g., Reviews, Testimonials):**
- Primary code: Create new component file in `src/components/` (e.g., `Reviews.tsx`)
- Data: If needed, extend `src/data/products.ts` or create `src/data/reviews.ts`
- Integration: Import and compose in `src/App.tsx`
- Styling: Use Tailwind utility classes and CSS variables from `src/index.css`

**New Component/Module:**
- Implementation: Add new `.tsx` file to `src/components/` with clear naming
- Props: Define TypeScript interface in same file above component export
- Pattern: Follow existing functional component pattern with React hooks

**Utilities/Helpers:**
- Shared helpers: Create `src/utils/` directory if multiple files needed
- Single file: Add functions to `src/data/` if data-related, otherwise create `src/utils/helperName.ts`
- Exports: Use named exports for functions, default export for components

**Styling/Theming:**
- New theme colors: Add CSS variables to `:root` in `src/index.css`
- Component-specific styles: Use Tailwind utility classes inline in JSX
- Global styles: Add to `src/index.css` (e.g., custom animations, typography)

**Images/Assets:**
- Product images: Place in `public/images/hero/` and reference as `/images/hero/filename`
- SVG components: Import into `src/assets/` for bundling or use inline SVG in JSX
- Favicon/metadata: Update `index.html`

## Special Directories

**node_modules/:**
- Purpose: Dependency installation location
- Generated: By `npm install` from package-lock.json
- Committed: No (.gitignore entry)
- Contains: React, Tailwind, Framer Motion, Vite, ESLint, TypeScript, and transitive deps

**dist/:**
- Purpose: Production build output
- Generated: By `npm run build` (Vite)
- Committed: No (.gitignore entry)
- Contains: Optimized and minified JavaScript, CSS, and assets ready for deployment

**.git/:**
- Purpose: Git version control repository
- Generated: By git initialization
- Committed: Yes (contains history and refs)

**.planning/codebase/:**
- Purpose: GSD codebase analysis documents
- Generated: By GSD mappers (ARCHITECTURE.md, STRUCTURE.md, CONVENTIONS.md, etc.)
- Committed: Typically yes, for team reference
- Contains: Architecture docs, testing patterns, concerns

## File Organization Best Practices

**Component Anatomy:**
```tsx
// Imports
import { useState } from 'react'
import { motion } from 'framer-motion'

// Interface (if applicable)
interface ComponentProps {
  title: string
}

// Component
export default function Component({ title }: ComponentProps) {
  const [state, setState] = useState('')

  return (
    <div>Content</div>
  )
}
```

**Props Drilling:**
- Keep props interfaces close to component definition
- Avoid deeply nested prop passing; consider refactoring into separate components
- Use callbacks to lift state up and pass down via props

**Asset Referencing:**
- Static images: Use `/images/` paths (served from `public/`)
- Imported SVGs: Import directly from `src/assets/`
- Images in product data: Store paths as strings in `src/data/products.ts`
